# Product Overview

## Problem statement

Most note apps force a tradeoff between quick capture and deep knowledge management. This project delivers both in a native Apple app while preserving user ownership of data.

## Target users

- Knowledge workers and developers in Apple ecosystem
- Users who want markdown portability and low lock-in
- Meeting-heavy users who need transcript-driven action capture
- Users curating knowledge via RSS feeds

## Platforms

- iOS (iPhone + iPad)
- macOS (native SwiftUI)

## Non-goals (v1)

- Multi-user collaboration and real-time co-editing
- Android/web parity
- Third-party cloud providers beyond iCloud
- Plugin marketplace/API platform

## Core workflows

1. Quick text note capture/editing in markdown
2. Connect notes using `[[wikilinks]]` and backlinks
3. Add RSS feeds and approve specific items into notes
4. Record meeting audio, transcribe, extract action items
5. Discover notes via full-text search and graph view

## Product principles

- Privacy first (local-first + iCloud under user account)
- Native Apple UX and platform integrations
- Progressive complexity (text-first, then advanced features)
- Portability (markdown + export)
